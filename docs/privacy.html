Privacy & Security Policy

Understand how PortSentry handles your data and keeps your dev scope secure.

1. 100% Local Data Processing

PortSentry monitors your active network ports and local CPU/memory usage telemetry. None of this data is collected, stored, or transmitted online.

  • No External Telemetry: PortSentry has no analytics scripts, logging servers, or usage trackers built-in.
  • No Cloud Storage: Any data shown in the graphical dashboard resides entirely in local system RAM and is cleared immediately upon window exit.
  • Complete Offline Capabilities: The application is designed to function fully on air-gapped developer workstations without requiring internet access.

2. Loopback Binding Security

The backend telemetry endpoint built inside the application is configured to listen strictly on the loopback network address (127.0.0.1 or localhost).

This binding ensures that no devices on your external Local Area Network (LAN), Wi-Fi router, or the public internet can query your process metrics or issue process termination requests to your machine.

3. Process Management Boundaries

PortSentry only terminates background processes (using Windows taskkill or Unix signals) when you explicitly click the Kill or Kill All buttons. The app will never automatically close or modify any running software on your machine without your direct interaction.

4. Open Source Audits

Because developer tooling handles sensitive command scopes, PortSentry is fully audit-ready. You can review the complete package definition and source code at GitHub.
